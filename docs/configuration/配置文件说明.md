# LiqPass 项目配置文件说明

## 项目根目录配置文件

### 1. `.gitignore`
**位置**: `/Users/zhaomosheng/Desktop/LiqPass-clean/.gitignore`
**作用**: Git版本控制忽略规则
**包含内容**:
- Python缓存和构建产物
- Node.js依赖和构建文件
- 操作系统和编辑器临时文件
- 环境变量和密钥文件
- 证据文件和日志文件

### 2. `.npmrc`
**位置**: `/Users/zhaomosheng/Desktop/LiqPass-clean/.npmrc`
**作用**: npm包管理器配置
**配置项**:
- `registry=https://registry.npmjs.org/` - npm仓库地址
- `fund=false` - 禁用fund提示
- `audit=false` - 禁用audit提示

### 3. `.nvmrc`
**位置**: `/Users/zhaomosheng/Desktop/LiqPass-clean/.nvmrc`
**作用**: Node.js版本管理
**指定版本**: v20.18.1 (LTS版本)
**兼容性**: 支持 v20.x, v22.x, v24.x 版本

### 4. `test-verification.sh`
**位置**: `/Users/zhaomosheng/Desktop/LiqPass-clean/test-verification.sh`
**作用**: 项目整体验证脚本
**功能**:
- 检查服务端口占用情况
- 测试各组件健康检查
- 验证订单验证API
- 检查证据文件和数据库

## 为什么这些文件在根目录？

这些配置文件必须放在项目根目录的原因：

1. **工具自动识别**: Git、npm、nvm等工具会自动在根目录查找配置
2. **全局作用**: 这些配置影响整个项目，包括所有子目录
3. **标准约定**: 符合软件开发的最佳实践
