# 发起赔付路由接口文档

## 基本信息
- **页面名称**: 发起赔付 (ClaimNewPage)
- **路由路径**: `/account/claims/new`
- **组件文件**: ClaimNewPage 组件
- **页面类型**: 用户管理页面
- **访问权限**: 已登录用户

## 路由配置
```typescript
<Route path="/account/claims/new" element={<ClaimNewPage />} />
```

## 页面功能
- 提供赔付申请表单界面
- 支持从订单发起赔付
- 展示赔付申请流程

## 接口定义

### 组件接口
```typescript
function ClaimNewPage(): JSX.Element
```

### 查询参数
- `order`: 订单ID（URL查询参数）

### 依赖组件
- `Crumb`: 面包屑导航组件
- `Section`: 区块容器组件
- `Card`: 卡片容器组件
- `useLocation`: React Router 位置钩子

## 页面布局
```typescript
<>
  <Crumb items={[
    {to:"/",label:"首页"},
    {to:"/account/claims",label:"我的赔付"},
    {label:"发起赔付"}
  ]}/>
  <Section title="发起赔付">
    <Card>
      <div className="text-sm text-stone-600">订单：<span className="font-mono">{order||'—'}</span></div>
      <div className="mt-2 text-xs text-stone-500">表单占位：上传凭证、说明原因、选择打款地址</div>
    </Card>
  </Section>
</>
```

## 面包屑导航
- 首页 → 我的赔付 → 发起赔付
- 支持返回上级页面

## 查询参数处理
```typescript
const loc = useLocation();
const qs = new URLSearchParams(loc.search);
const order = qs.get('order');
```

## 表单功能规划
- 上传凭证文件
- 说明赔付原因
- 选择打款地址
- 提交赔付申请

## 国际化文本
- 页面标题: "发起赔付"
- 面包屑: 首页 → 我的赔付 → 发起赔付
- 区块标题: "发起赔付"
- 订单标签: "订单："

## 响应式设计
- 使用 `Section` 组件管理区块布局
- 响应式内边距和间距
- 支持移动端表单操作

## 技术特点
- 支持URL查询参数传递订单信息
- 使用 `Section` 组件统一区块样式
- 表单功能占位，可扩展完整实现

## 数据流设计
- 从URL参数获取关联订单
- 支持独立赔付申请（无订单关联）
- 表单数据提交到后端API

## 扩展功能

- 表单验证和错误处理
- 赔付类型选择
- 赔付金额计算

## 注意事项
- 当前为表单占位实现
- 需要完善表单字段和验证
- 可扩展为多步骤表单流程
