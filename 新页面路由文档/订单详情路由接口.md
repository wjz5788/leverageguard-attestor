# 订单详情路由接口文档

## 基本信息
- **页面名称**: 订单详情 (OrderDetailPage)
- **路由路径**: `/account/orders/:id`
- **组件文件**: OrderDetailPage 组件
- **页面类型**: 用户管理页面
- **访问权限**: 已登录用户

## 路由配置
```typescript
<Route path="/account/orders/:id" element={<OrderDetailPage />} />
```

## 页面功能
- 显示特定订单的详细信息
- 展示订单的链上事件记录
- 提供订单状态跟踪功能

## 接口定义

### 组件接口
```typescript
function OrderDetailPage(): JSX.Element
```

### 路由参数
- `id`: 订单ID（动态路由参数）

### 依赖组件
- `Crumb`: 面包屑导航组件
- `Card`: 卡片容器组件
- `useParams`: React Router 参数钩子

## 页面布局
```typescript
<>
  <Crumb items={[
    {to:"/",label:"首页"},
    {to:"/account/orders",label:"我的订单"},
    {label:String(id)}
  ]}/>
  <section className="mx-auto max-w-7xl px-4 py-12">
    <Card>
      <div className="text-sm text-stone-600">链上事件：</div>
      <ul className="mt-2 list-disc pl-6 text-sm">
        <li>OrderCreated …</li>
        <li>PaymentConfirmed …</li>
      </ul>
    </Card>
  </section>
</>
```

## 面包屑导航
- 首页 → 我的订单 → 订单ID
- 支持返回上级页面

## 链上事件展示
- OrderCreated: 订单创建事件
- PaymentConfirmed: 支付确认事件
- 使用列表形式展示事件记录

## 国际化文本
- 页面标题: "订单详情"
- 面包屑: 首页 → 我的订单 → {订单ID}
- 事件标题: "链上事件："

## 响应式设计
- 使用 `max-w-7xl` 容器限制最大宽度
- 响应式内边距: `px-4 py-12`
- 列表使用标准缩进格式

## 技术特点
- 使用动态路由参数获取订单ID
- 静态事件数据展示
- 可扩展为从区块链API获取真实事件数据

## 数据扩展性
- 可添加更多链上事件类型
- 支持事件时间戳显示
- 可集成交易哈希链接

## 注意事项
- 当前使用静态示例数据
- 需要订单ID参数才能正常显示
- 可扩展为实时获取链上数据
