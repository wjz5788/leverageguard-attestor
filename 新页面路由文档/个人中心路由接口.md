# P0 · 必需页

## 1) 购买下单

* 路由：`/buy`
* 目的：选 SKU（24h/8h/月度/无爆仓返现），输入本金与杠杆，用 Base USDC 下单。
* 元素：`#sel-sku` `#in-principal` `#in-leverage` `#btn-preview` `#btn-create`
* 依赖：钱包 S3（已连 + Base）；USDC 余额；价单预览。
* 接口：`GET /catalog/skus` → 列表；`POST /orders/preview` → 报价；`POST /orders` → 下单（返回 `orderId`）。

## 2) 交易所 API 只读校验（已定义）

* 路由：`/settings/exchange-api`
* 目的：录入并测试 Binance/OKX 只读密钥，生成 `verifyKey`。
* 接口：`POST /verify/exchange/test` → 返回 `verifyKey`；`POST /verify/exchange/dispose`

## 3) 赔付（取证→判定→链上）

* 路由：`/claims/payout`
* 目的：输入订单号，基于 `verifyKey` 预审赔付金额，提交赔付并上链取证。
* 元素：`#in-order-ref` `#sel-exchange` `#btn-preview` `#btn-submit` `#btn-status` `#btn-download-proof`
* 依赖：钱包 S3；有效 `verifyKey`；Base 原生 USDC。
* 接口：`POST /claims/preview` → `{eligible,payoutUSDC,reasons,idempotencyKey}`；`POST /claims/submit` → `{claimId}`；`POST /claims/{claimId}/payout`；`GET /claims/{claimId}/status`；`GET /claims/{claimId}/evidence`；`GET /attestations/{claimId}`

## 4) 赔付历史

* 路由：`/claims/history`
* 目的：展示本人所有理赔单与状态、交易哈希、可下载凭证。
* 元素：筛选、表格、状态徽标。
* 接口：`GET /claims?wallet=<addr>`；`GET /claims/{claimId}/status`

## 5) 透明性（上链公示）

* 路由：`/transparency`
* 目的：公开 Merkle 根、Attestation、合约事件；可按钱包/时间筛选。
* 接口：`GET /attestations/search?wallet?&from?&to?`；`GET /attestations/{claimId}`

# P1 · 运行与配置

## 6) 基础设置 · 钱包与网络

* 路由：`/settings/wallet`
* 目的：连接 MetaMask、强制 Base、添加原生 USDC。
* 元素：`#btn-connect` `#btn-switch-base` `#btn-add-usdc` `#out-account` `#out-chain`
* 无服务端接口（仅链交互）。

## 7) 订单历史

* 路由：`/orders/history`
* 目的：查看本人下单记录与状态、对应 `orderRef`。
* 接口：`GET /orders?wallet=<addr>`；`GET /orders/{orderId}`

## 8) 帮助与指引

* 路由：`/help`
* 目的：新手流程、权限说明、失败码对照、视频/截图。
* 无强制接口。

## 9) 系统状态

* 路由：`/status`
* 目的：汇总 US-backend / JP-verify / 链 RPC 健康度。
* 接口：`GET /healthz`（US）；`GET /verify/healthz`（JP）；链 RPC ping

# P1 · 管理面（仅内用）

## 10) 证据导入

* 路由：`/admin/evidence/import`
* 目的：导入本地目录或文件，标准化、去重、补齐 merkleRoot，可选自动上链。
* 接口：`POST /admin/evidence/import`；`POST /admin/evidence/{id}/onchain`；`GET /admin/evidence/{id}`

## 11) 理赔审核队列

* 路由：`/admin/claims`
* 目的：人工复核 `admin_pending` 单，允许或拒绝，重试上链。
* 接口：`GET /admin/claims?status=admin_pending`；`POST /admin/claims/{id}/approve`；`POST /admin/claims/{id}/reject`

## 12) 风控与参数

* 路由：`/admin/settings`
* 目的：黑名单、冷却期、SKU 开关、`AUTO_ONCHAIN` / `SIMULATED_PAYOUT`。
* 接口：`GET/POST /admin/settings`

# P2 · 法务与杂项

## 13) 法务

* 路由：`/legal`
* 目的：ToS、隐私、免责声明、地区限制。

## 14) 登录页/404

* 路由：`/404`
* 目的：统一异常与重定向。

# 跨页共享模块（AI 需实现）

* **钱包状态机**：S0 未安装 → S1 未连接 → S2 非 Base → S3 就绪；事件 `wallet:*` 广播。
* **会话横幅**：展示 `verifyKey` 剩余时间，超时引导回 `/settings/exchange-api`。
* **请求拦截**：自动附带 `wallet` 地址与 `verifyKey`（若存在）；重放 `idempotencyKey`。
* **错误码映射**：统一文案集，用于 Toast 与表格徽标。
* **区块浏览器链接**：Base 主网 tx/address 跳转。

# 推荐实施顺序

1. `/settings/wallet` → `/settings/exchange-api`（拿到 `verifyKey`）
2. `/buy`（下单）
3. `/claims/payout` + `/claims/history`
4. `/transparency`
5. `/admin/*` 与 `/status`

# 每页验收要点（简）

* **/buy**：报价≤1s；创建单返回 `orderId`；钱包未在 Base 时禁用提交。
* **/settings/exchange-api**：返回 `verifyKey`；失败码可视化。
* **/claims/payout**：不可赔给出明确 `reasons`；赔付与上链哈希可点击。
* **/claims/history**：分页与状态轮询。
* **/transparency**：按钱包过滤；展示 Merkle/事件。
* **/admin/evidence/import**：重复导入幂等，`onchain` 可重试。
